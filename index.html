Dynamics of Team Coordination

Usage of Kanban Board

We used GitHub projects to see current status for each task. Every issue had a full description, start date and deadline.

Communication Within the Team

For daily communication we used:

  • Discord:
    • personal chats
    • group chat
  • GitHub:
    • comments in pull requests
  • Due to Kanban board, Discord and Github we knew the progress of every task for every team member.
  • It was enough for our team to work smoothly and solve current issues at any time of the day or night.
  • So we didn't use daily calls.

Instances of successful communication

Approach to Task Allocation

  1. Adding all tasks to the Kanban board
  2. Defining the start and end times of tasks
  3. Everyone chooses tasks for his chose
  4. Real-time update of work in progress
  5. Daily task status tracking
  6. If tasks overlap, close communication with other team members
  7. Creation of a PR with a detailed description of the completed task and screenshots
  8. Every PR should be reviewed and approved at least one team member
